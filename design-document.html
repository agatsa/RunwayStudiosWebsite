1 Product Overview

Runway Studios AI Growth OS is a SaaS marketing intelligence platform built for Indian D2C brands and digital agencies. It aggregates advertising performance data from multiple platforms — Meta Ads, Google Ads, and YouTube — into a unified analytics dashboard, and applies AI (Claude by Anthropic) to generate cross-channel performance insights and prioritised optimisation recommendations.

The platform is hosted on Google Cloud Run (asia-south1) and serves clients through a web dashboard at dashboard.runwaystudios.co. All Google Ads data is accessed via the Google Ads API exclusively for read-only reporting purposes.

Core principle: The Google Ads API is used solely for reading performance data. The application does not create, modify, pause, enable, or delete any Google Ads campaigns, ad groups, ads, or keywords.

Token Storage

  • Access tokens and refresh tokens stored in encrypted columns in PostgreSQL (Cloud SQL, asia-south1)
  • Tokens are workspace-scoped — each client's tokens are isolated by workspace_id
  • Refresh tokens are used automatically when access tokens expire (Google's standard 1-hour expiry)
  • Users can revoke access at any time via Google Account settings or the platform's Settings page

4 Google Ads API Usage

All API calls are made using GAQL (Google Ads Query Language) SELECT queries. No mutate operations are performed.

Report Type GAQL Resource Fields Retrieved Purpose
Campaign Performance campaign name, status, impressions, clicks, cost, conversions, conversion_value Dashboard KPI cards and spend charts
Keyword Performance keyword_view keyword text, match type, quality score, impressions, clicks, cost, conversions Keyword analysis, QS badges, wasted spend detection
Search Terms search_term_view search term, campaign, clicks, cost, conversions Negative keyword recommendations, intent analysis
Geographic Performance geographic_view location, campaign, impressions, clicks, cost, conversions City/region ROAS breakdown, expansion opportunities
Device Performance campaign (device segment) device, campaign, impressions, clicks, cost, conversions Mobile vs desktop budget optimisation
Time of Day campaign (hour segment) hour of day, day of week, cost, conversions Ad scheduling recommendations heatmap
Auction Insights auction_insight domain, impression share, overlap rate, position above rate Competitive position analysis

Account List CustomerService.listAccessibleCustomers customer_id, descriptive_name Multi-account discovery for agency users

No mutate operations are used. The application does not call CampaignService.mutate, AdGroupService.mutate, BudgetService.mutate, or any other write operation. All interactions with the Google Ads API are strictly SELECT queries via GAQL.

5 Data Flow & Storage

Ingestion Pipeline

  1. User authenticates via OAuth2 and grants access to their Google Ads account
  2. The FastAPI backend calls Google Ads API using the stored refresh token
  3. Performance metrics are retrieved for the requested date range via GAQL SELECT queries
  4. Data is normalised and stored in the kpi_hourly table in PostgreSQL, keyed by workspace_id, platform, entity_level, and hour_ts
  5. The Next.js dashboard reads this data via internal REST API endpoints
  6. Claude AI analyses the aggregated data and returns text recommendations (not stored, generated per request)

Data Retention & Isolation

6 Multi-Account (Agency) Support

The platform is designed for agencies managing multiple client Google Ads accounts. Each client account is represented as a separate Workspace in the platform. The architecture follows Google's recommended manager account (MCC) model:

  • A single OAuth2 token from the agency's Google account (with MCC access) can access multiple client accounts
  • CustomerService.listAccessibleCustomers is called once to discover available customer IDs
  • Each subsequent GAQL query specifies the target login-customer-id header to scope results to the correct client account
  • Client data is strictly isolated by workspace_id — no cross-client data leakage is possible
ScenarioImplementation
Brand managing own account Direct OAuth2 with their Google account. Single customer_id.
Agency managing client accounts OAuth2 with agency's MCC account. Per-client login-customer-id in each request.
